# Lumiere-Lang

A small programming language that compiles to native code via LLVM.

## Language Features

- Integer literals
- Variable declarations with `let`
- Arithmetic operators: `+`, `-`, `*` (with correct precedence)
- `print` to output values

## Prerequisites

- OCaml 5.x with opam
- [Menhir](https://gallium.inria.fr/~fpottier/menhir/) parser generator (`opam install menhir`)
- LLVM with clang (`brew install llvm` on macOS)

## Build

```sh
dune build
```

## Example

Create a file `examples/hello.lum`:

```
let a = 1
let b = 2 + 3 + 4
print (a + b)
```

Compile and run it:

```sh
$ dune exec bin/lumiere.exe -- examples/hello.lum
Compiled to examples/hello

$ ./examples/hello
10
```

Use `--emit-llvm` to inspect the generated LLVM IR:

```sh
$ dune exec bin/lumiere.exe -- examples/hello.lum --emit-llvm
; Generated by Lumiere compiler

@.fmt = private unnamed_addr constant [4 x i8] c"%d\0A\00"

declare i32 @printf(ptr, ...)

define i32 @main() {
entry:
  %t0 = add i32 0, 1
  %t1 = add i32 0, 2
  %t2 = add i32 0, 3
  %t3 = add i32 %t1, %t2
  %t4 = add i32 0, 4
  %t5 = add i32 %t3, %t4
  %t6 = add i32 %t0, %t5
  call i32 (ptr, ...) @printf(ptr @.fmt, i32 %t6)
  ret i32 0
}
```

## How It Works

```
Source (.lum) → Lexer (ocamllex) → Parser (menhir) → AST → Codegen → LLVM IR → clang → Native Binary
```

1. **Lexer** (`lib/lexer.mll`) — tokenizes the source into keywords, identifiers, integers, and operators.
2. **Parser** (`lib/parser.mly`) — parses tokens into an AST with correct operator precedence.
3. **Codegen** (`lib/codegen.ml`) — walks the AST and emits LLVM IR as text.
4. **Driver** (`bin/lumiere.ml`) — orchestrates the pipeline and invokes clang to produce a native binary.
